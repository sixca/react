값타입 변수 - 값을 저장한다
참조타입 변수 - 값이 있는 곳의 주소(참조)를 저장한다
프로그램을 가동하면 할당되는 메모리가 3부분이다
static, stack, heap 공간임(논리적 단위이다)
static에는 static 메서드나 함수, 상수 등이 저장된다(시스템이 관리)

stack - 함수 호출 시 호출된 함수가 차지하는 메모리, 들어간 순서와 반대로 나오는 구조이다.
        일반변수들은 이 공간에 확보된다
int a; // 값 변수. 스택공간에 자리잡고 이 공간에 확보된다.
a = 9; 9라는 상수값이 a메모리에 저장된다.
스택에 대한 모든 관리는 시스템이 담당한다.

heap - 힙(free store라고도 한다)은 객체나 배열이 저장된다.
이 공간에 메모리를 확보하고 데이터를 저장하면 이 공간에 대한 주소를 반환한다

Person p1; //변수 자체는 스택에 저장된다. 이 변수에는 객체의 주소가 저장되기 대문에
참조변수라고 부른다. 아직 데이터를 저장할 공간은 없다.

let p1 = new person(); 힙공간에 person 객체를 생성하고 생성한 객체의 첫번째 번지값을 p1에 할당한다.
만일 메모리가 부족해서 객체 생성을 못할 경우 null로 채워진다.

p1 = new Person(); 앞에 객체는 참조하는 변수가 없으면 garbage collector(gc)가 메모리 부족할 때 수거를 해간다.

List<Integer> list = new ArrayList<Integer>();   //자바 배열 선언 ::  integer 객체들을 add하여 동적으로 사용할 수 있다. (list.add(1,10);)
값 타입을 객체로 전환해야 할 때가 있다. 
값 => 객체로 전환시키는 클래스를 Wrapper Class라고 한다.(박싱, 언박싱)
(ex) 값타입이 참조타입으로 전환될 때, 참조타입이 값으로 전환될때.
